    [问与答] shadowsocks 为什么不能自动监听 git http 请求?
    最近发现在 git 客户端以外的场合下, git clone 等操作通通连接失败. 搜索后得到解决方案: git config --global https.https://github.com.proxy https://127.0.0.1:xxxx 测试后发现有效. 搜了下 shadowsocks 的原理, 说是本地会建立名为 ss-local 的 socks5 代理(应该就是上面的https://127.0.0.1:xxxx), 并修改系统和浏览器等配置, 从而将本地流量全部导向该 socks 代理, 之后再加密发送给 ss-server. 于是我很好奇, 同样走 http, 为什么 git 的请求没有被自动转发呢, 还必须得手动设置.
